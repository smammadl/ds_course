# Stochastic Gradient Descent (SGD)

Stochastic Gradient Descent (SGD) is one of the most fundamental optimization algorithms in machine learning. It's used to minimize the loss function by iteratively moving in the direction of the steepest descent.

## Basic Concept

The core idea of SGD is to update model parameters in the opposite direction of the gradient of the loss function with respect to those parameters.

## Mathematical Formulation

### Standard Gradient Descent Update Rule

The basic update rule for gradient descent is:

$$
\theta_{t+1} = \theta_t - \eta \nabla_{\theta} J(\theta_t)
$$

Where:

- $\theta_t$ represents the model parameters at iteration $t$
- $\eta$ is the learning rate (step size)
- $\nabla_{\theta} J(\theta_t)$ is the gradient of the loss function $J$ with respect to the parameters $\theta$ at iteration $t$

### Stochastic vs. Batch Gradient Descent

#### Batch Gradient Descent

In batch gradient descent, we compute the gradient using the entire training dataset:

$$
\nabla_{\theta} J(\theta) = \frac{1}{n} \sum_{i=1}^{n} \nabla_{\theta} J_i(\theta)
$$
$$
\theta \leftarrow \theta - \eta \nabla_{\theta} J(\theta)
$$

Where $J_i(\theta)$ is the loss for the $i$-th training example.

#### Stochastic Gradient Descent

In stochastic gradient descent, we update the parameters using only one training example at a time:

$$
\theta_{t+1} = \theta_t - \eta \nabla_{\theta} J_i(\theta_t)
$$

Where $i$ is randomly selected for each update.

#### Mini-Batch Gradient Descent

In practice, we often use mini-batch SGD, which is a compromise between the two:

$$
\theta_{t+1} = \theta_t - \eta \frac{1}{|B|} \sum_{i \in B} \nabla_{\theta} J_i(\theta_t)
$$

Where $B$ is a mini-batch of examples randomly sampled from the training set.

### SGD with Momentum

SGD can be enhanced with momentum, which helps accelerate convergence and dampen oscillations:

$$
\nu_{t+1} = \gamma \nu_t + \eta \nabla_{\theta} J(\theta_t)
$$
$$
\theta_{t+1} = \theta_t - \nu_{t+1}
$$

Where:

- $\nu_t$ is the velocity vector (initially $\nu_0 = 0$)
- $\gamma$ is the momentum coefficient (typically between 0 and 1, for example 0.9)

This looks ahead by calculating the gradient at the approximate future position.

### SGD with Nesterov Momentum

A further refinement is Nesterov accelerated gradient (NAG):

$$
\nu_{t+1} = \gamma \nu_t + \eta \nabla_{\theta} J(\theta_t - \gamma \nu_t)
$$

$$
\theta_{t+1} = \theta_t - \nu_{t+1}
$$

This looks ahead by calculating the gradient at the approximate future position.

## Advantages and Disadvantages

#### Advantages

- Simple to implement and understand
- Low memory requirements
- Works well for sparse data when combined with appropriate regularization

#### Disadvantages

- Requires careful tuning of learning rate
- May get trapped in local minima or saddle points
- Can be slow to converge compared to adaptive methods like Adam
- Sensitive to feature scaling

SGD remains a popular choice for many deep learning applications, especially when combined with momentum and proper learning rate scheduling.

## Simple Example: Minimizing a Scalar Function

Let's walk through a concrete example of using SGD to minimize a simple scalar function:

$$f(x) = x^2 + 10$$

This is a simple parabola with a global minimum at $x = 0$, where $f(0) = 10$.

The derivative of this function is:

$$f'(x) = 2x$$

Now let's apply SGD to find the minimum:

### Initial Setup
- Starting point: $x_0 = 5$
- Learning rate: $\eta = 0.1$
- We'll run for 10 iterations

### Iteration 1
- Current position: $x_0 = 5$
- Gradient: $f'(x_0) = 2 \times  5 = 10$
- Update: $x_1 =  x_0 - \eta \times f'(x_0) =  5 - 0.1 \times 10 = 5 - 1 = 4$
- New function value: $f(x_1) = 4^2 + 10 = 16 + 10 = 26$

### Iteration 2
- Current position: $x_1 = 4$
- Gradient: $f'(x_1) = 2 \times  4 = 8$
- Update: $x_2 =  x_1 - \eta \times f'(x_1) =  4 - 0.1 \times 8 = 4 - 0.8 = 3.2$
- New function value: $f(x_2) = 3.2^2 + 10 = 10.24 + 10 = 20.24$

### Iteration 3
- Current position: $x_2 = 3.2$
- Gradient: $f'(x_2) = 2 \times  3.2 = 6.4$
- Update: $x_3 =  x_2 - \eta \times f'(x_2) =  3.2 - 0.1 \times 6.4 = 3.2 - 0.64 = 2.56$
- New function value: $f(x_3) = 2.56^2 + 10 = 6.55 + 10 = 16.55$

### Iteration 4
- Current position: $x_3 = 2.56$
- Gradient: $f'(x_3) = 2 \times 2.56 = 5.12$
- Update: $x_4 = x_3 - \eta \times f'(x_3) = 2.56 - 0.1 \times 5.12 = 2.56 - 0.512 = 2.048$
- New function value: $f(x_4) = 2.048^2 + 10 = 4.19 + 10 = 14.19$

### Iteration 5
- Current position: $x_4 = 2.048$
- Gradient: $f'(x_4) = 2 \times 2.048 = 4.096$
- Update: $x_5 = x_4 - \eta \times f'(x_4) = 2.048 - 0.1 \times 4.096 = 2.048 - 0.4096 = 1.6384$
- New function value: $f(x_5) = 1.6384^2 + 10 = 2.68 + 10 = 12.68$

### Summary of Progress

| Iteration | x value | Gradient | Function Value |
|-----------|---------|----------|----------------|
| 0         | 5.0000  | 10.0000  | 35.0000        |
| 1         | 4.0000  | 8.0000   | 26.0000        |
| 2         | 3.2000  | 6.4000   | 20.2400        |
| 3         | 2.5600  | 5.1200   | 16.5536        |
| 4         | 2.0480  | 4.0960   | 14.1943        |
| 5         | 1.6384  | 3.2768   | 12.6829        |

If we continued for more iterations, $x$ would approach 0 (the true minimum), and the function value would approach 10.

## Example: SGD with Momentum

Let's extend our example to use momentum, which helps accelerate SGD in relevant directions and dampens oscillations. We'll use the same function $f(x) = x^2 + 10$ with its derivative $f'(x) = 2x$.

### Initial Setup
- Starting point: $x_0 = 5$
- Learning rate: $\eta = 0.1$
- Momentum coefficient: $\gamma = 0.9$
- Initial velocity: $v_0 = 0$
- We'll run for 5 iterations

### Iteration 1
- Current position: $x_0 = 5$
- Gradient: $f'(x_0) = 2 \times 5 = 10$
- Update velocity: $v_1 = \gamma v_0 + \eta \times f'(x_0) = 0.9 \times 0 + 0.1 \times 10 = 1$
- Update position: $x_1 = x_0 - v_1 = 5 - 1 = 4$
- New function value: $f(x_1) = 4^2 + 10 = 26$

### Iteration 2
- Current position: $x_1 = 4$
- Gradient: $f'(x_1) = 2 \times 4 = 8$
- Update velocity: $v_2 = \gamma v_1 + \eta \times f'(x_1) = 0.9 \times 1 + 0.1 \times 8 = 0.9 + 0.8 = 1.7$
- Update position: $x_2 = x_1 - v_2 = 4 - 1.7 = 2.3$
- New function value: $f(x_2) = 2.3^2 + 10 = 15.29$

### Iteration 3
- Current position: $x_2 = 2.3$
- Gradient: $f'(x_2) = 2 \times 2.3 = 4.6$
- Update velocity: $v_3 = \gamma v_2 + \eta \times f'(x_2) = 0.9 \times 1.7 + 0.1 \times 4.6 = 1.53 + 0.46 = 1.99$
- Update position: $x_3 = x_2 - v_3 = 2.3 - 1.99 = 0.31$
- New function value: $f(x_3) = 0.31^2 + 10 = 10.10$

### Iteration 4
- Current position: $x_3 = 0.31$
- Gradient: $f'(x_3) = 2 \times 0.31 = 0.62$
- Update velocity: $v_4 = \gamma v_3 + \eta \times f'(x_3) = 0.9 \times 1.99 + 0.1 \times 0.62 = 1.791 + 0.062 = 1.853$
- Update position: $x_4 = x_3 - v_4 = 0.31 - 1.853 = -1.543$
- New function value: $f(x_4) = (-1.543)^2 + 10 = 12.38$

### Iteration 5
- Current position: $x_4 = -1.543$
- Gradient: $f'(x_4) = 2 \times (-1.543) = -3.086$
- Update velocity: $v_5 = \gamma v_4 + \eta \times f'(x_4) = 0.9 \times 1.853 + 0.1 \times (-3.086) = 1.6677 - 0.3086 = 1.359$
- Update position: $x_5 = x_4 - v_5 = -1.543 - 1.359 = -2.902$
- New function value: $f(x_5) = (-2.902)^2 + 10 = 18.42$

### Summary of Progress with Momentum

| Iteration | x value  | Gradient | Velocity | Function Value |
|-----------|----------|----------|----------|----------------|
| 0         | 5.0000   | 10.0000  | 0.0000   | 35.0000        |
| 1         | 4.0000   | 8.0000   | 1.0000   | 26.0000        |
| 2         | 2.3000   | 4.6000   | 1.7000   | 15.2900        |
| 3         | 0.3100   | 0.6200   | 1.9900   | 10.0961        |
| 4         | -1.5430  | -3.0860  | 1.8530   | 12.3809        |
| 5         | -2.9020  | -5.8040  | 1.3591   | 18.4216        |

Notice how momentum causes the algorithm to overshoot the minimum (at x=0) and then oscillate around it. With more iterations and proper tuning of the momentum coefficient, it would eventually converge.

## Example: SGD with Nesterov Momentum

Now let's implement SGD with Nesterov momentum, which improves upon standard momentum by calculating the gradient at an approximated future position.

### Initial Setup
- Starting point: $x_0 = 5$
- Learning rate: $\eta = 0.1$
- Momentum coefficient: $\gamma = 0.9$
- Initial velocity: $v_0 = 0$
- We'll run for 5 iterations

### Iteration 1
- Current position: $x_0 = 5$
- Look-ahead position: $x_0 - \gamma v_0 = 5 - 0.9 \times 0 = 5$
- Gradient at look-ahead: $f'(5) = 2 \times 5 = 10$
- Update velocity: $v_1 = \gamma v_0 + \eta \times f'(5) = 0.9 \times 0 + 0.1 \times 10 = 1$
- Update position: $x_1 = x_0 - v_1 = 5 - 1 = 4$
- New function value: $f(x_1) = 4^2 + 10 = 26$

### Iteration 2
- Current position: $x_1 = 4$
- Look-ahead position: $x_1 - \gamma v_1 = 4 - 0.9 \times 1 = 4 - 0.9 = 3.1$
- Gradient at look-ahead: $f'(3.1) = 2 \times 3.1 = 6.2$
- Update velocity: $v_2 = \gamma v_1 + \eta \times f'(3.1) = 0.9 \times 1 + 0.1 \times 6.2 = 0.9 + 0.62 = 1.52$
- Update position: $x_2 = x_1 - v_2 = 4 - 1.52 = 2.48$
- New function value: $f(x_2) = 2.48^2 + 10 = 16.15$

### Iteration 3
- Current position: $x_2 = 2.48$
- Look-ahead position: $x_2 - \gamma v_2 = 2.48 - 0.9 \times 1.52 = 2.48 - 1.368 = 1.112$
- Gradient at look-ahead: $f'(1.112) = 2 \times 1.112 = 2.224$
- Update velocity: $v_3 = \gamma v_2 + \eta \times f'(1.112) = 0.9 \times 1.52 + 0.1 \times 2.224 = 1.368 + 0.2224 = 1.5904$
- Update position: $x_3 = x_2 - v_3 = 2.48 - 1.5904 = 0.8896$
- New function value: $f(x_3) = 0.8896^2 + 10 = 10.79$

### Iteration 4
- Current position: $x_3 = 0.8896$
- Look-ahead position: $x_3 - \gamma v_3 = 0.8896 - 0.9 \times 1.5904 = 0.8896 - 1.43136 = -0.54176$
- Gradient at look-ahead: $f'(-0.54176) = 2 \times (-0.54176) = -1.08352$
- Update velocity: $v_4 = \gamma v_3 + \eta \times f'(-0.54176) = 0.9 \times 1.5904 + 0.1 \times (-1.08352) = 1.43136 - 0.108352 = 1.323008$
- Update position: $x_4 = x_3 - v_4 = 0.8896 - 1.323008 = -0.433408$
- New function value: $f(x_4) = (-0.433408)^2 + 10 = 10.19$

### Iteration 5
- Current position: $x_4 = -0.433408$
- Look-ahead position: $x_4 - \gamma v_4 = -0.433408 - 0.9 \times 1.323008 = -0.433408 - 1.1907072 = -1.6241152$
- Gradient at look-ahead: $f'(-1.6241152) = 2 \times (-1.6241152) = -3.2482304$
- Update velocity: $v_5 = \gamma v_4 + \eta \times f'(-1.6241152) = 0.9 \times 1.323008 + 0.1 \times (-3.2482304) = 1.1907072 - 0.32482304 = 0.86588416$
- Update position: $x_5 = x_4 - v_5 = -0.433408 - 0.86588416 = -1.29929216$
- New function value: $f(x_5) = (-1.29929216)^2 + 10 = 11.69$

### Summary of Progress with Nesterov Momentum

| Iteration | x value     | Look-ahead Position | Gradient at Look-ahead | Velocity   | Function Value |
|-----------|-------------|---------------------|------------------------|------------|----------------|
| 0         | 5.0000      | 5.0000              | 10.0000                | 0.0000     | 35.0000        |
| 1         | 4.0000      | 3.1000              | 6.2000                 | 1.0000     | 26.0000        |
| 2         | 2.4800      | 1.1120              | 2.2240                 | 1.5200     | 16.1504        |
| 3         | 0.8896      | -0.5418             | -1.0835                | 1.5904     | 10.7914        |
| 4         | -0.4334     | -1.6241             | -3.2482                | 1.3230     | 10.1878        |
| 5         | -1.2993     | -2.0786             | -4.1572                | 0.8659     | 11.6882        |

Notice how Nesterov momentum also overshoots but tends to correct more quickly than standard momentum due to the look-ahead gradient calculation.

## Comparison of Convergence

If we plot the convergence of all three methods (basic SGD, SGD with momentum, and SGD with Nesterov momentum) for our simple function, we would observe:

1. **Basic SGD**: Steady, predictable convergence toward the minimum
2. **SGD with Momentum**: Faster initial progress but may overshoot and oscillate
3. **SGD with Nesterov Momentum**: Similar to momentum but with better correction after overshooting

For complex loss landscapes in deep learning:
- Momentum helps escape shallow local minima and plateaus
- Nesterov momentum provides additional responsiveness to changes in the gradient
- Both variants typically converge faster than basic SGD, especially for ill-conditioned problems

The choice between these variants depends on the specific optimization problem, with Nesterov momentum generally preferred when available due to its improved convergence properties.
