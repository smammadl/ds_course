# FTRL (Follow The Regularized Leader) Optimizer

FTRL, or Follow The Regularized Leader, is an online learning algorithm that was popularized by Google in the 2013 paper "Ad Click Prediction: a View from the Trenches" by McMahan et al. It was designed specifically for large-scale online learning problems, particularly those with sparse features, such as click-through rate prediction in online advertising.

## Key Idea

The key insight behind FTRL is to combine the benefits of two optimization approaches:
1. Online gradient descent for efficient updates
2. L1 regularization for feature selection and sparsity

FTRL is particularly effective at producing sparse models, which is crucial for deployment in memory-constrained environments or when feature selection is important.

## Mathematical Formulation

The FTRL-Proximal algorithm, which is the most commonly used variant, uses the following update rule:

### 1. Compute the Sum of Squared Gradients

$$\sigma_t = \sigma_{t-1} + g_t^2$$

Where:
- $\sigma_t$ is the sum of squared gradients up to time $t$
- $g_t$ is the gradient at time step $t$

### 2. Compute the Learning Rate for Each Parameter

$$\eta_t = \frac{\alpha}{\beta + \sqrt{\sigma_t}}$$

Where:
- $\alpha$ is the learning rate
- $\beta$ is a smoothing parameter (typically a small value like 1.0)

### 3. Compute the "z" Value

$$z_t = z_{t-1} + g_t - \sigma_t \cdot w_{t-1}$$

Where:
- $z_t$ is an accumulated value that helps determine the sign of the parameter
- $w_{t-1}$ is the parameter value at the previous time step

### 4. Parameter Update with L1 and L2 Regularization

$$w_t = 
\begin{cases} 
0 & \text{if } |z_t| \leq \lambda_1 \\
-\eta_t \cdot (z_t - \text{sign}(z_t) \cdot \lambda_1) & \text{otherwise}
\end{cases}$$

Where:
- $\lambda_1$ is the L1 regularization parameter
- $\text{sign}(z_t)$ returns the sign of $z_t$ (+1 or -1)

This update rule has several important properties:
1. It naturally produces sparse models due to the L1 regularization
2. It adapts the learning rate for each parameter based on historical gradients
3. It has good theoretical guarantees for online learning

## Advantages of FTRL

1. **Sparsity**: FTRL naturally produces sparse models, which is beneficial for memory efficiency and feature selection.

2. **Online Learning**: It's designed for online learning scenarios where data arrives sequentially.

3. **Adaptive Learning Rates**: The algorithm adapts learning rates per-parameter based on historical gradients.

4. **Regularization Control**: It provides fine-grained control over L1 and L2 regularization.

5. **Proven in Production**: FTRL has been successfully used in large-scale production systems, particularly in online advertising.

## When to Use FTRL

FTRL is particularly well-suited for:

- Large-scale online learning problems
- Scenarios where model sparsity is important
- Problems with high-dimensional, sparse feature spaces
- Click-through rate prediction and other online advertising tasks
- When memory efficiency is a concern

## Numerical Example: Minimizing a Simple Function with FTRL

Let's walk through a concrete example of using FTRL to minimize the simple function:

$$f(x) = x^2 + 10$$

This is a simple parabola with a global minimum at $x = 0$, where $f(0) = 10$.

The derivative of this function is:

$$f'(x) = 2x$$

Now let's apply FTRL to find the minimum:

### Initial Setup
- Starting point: $x_0 = 5$
- Learning rate: $\alpha = 0.1$
- Beta: $\beta = 1.0$
- L1 regularization: $\lambda_1 = 0.1$
- Initial sum of squared gradients: $\sigma_0 = 0$
- Initial z value: $z_0 = 0$
- Time step: $t = 0$

### Iteration 1 (t = 1)
- Current position: $x_0 = 5$
- Gradient: $g_1 = f'(x_0) = 2 \times 5 = 10$
- Update sum of squared gradients: $\sigma_1 = \sigma_0 + g_1^2 = 0 + 10^2 = 100$
- Compute learning rate: $\eta_1 = \frac{\alpha}{\beta + \sqrt{\sigma_1}} = \frac{0.1}{1.0 + \sqrt{100}} = \frac{0.1}{11} \approx 0.0091$
- Update z value: $z_1 = z_0 + g_1 - \sigma_1 \cdot x_0 = 0 + 10 - 100 \times 5 = 10 - 500 = -490$
- Since $|z_1| = 490 > \lambda_1 = 0.1$, we update $x_1$:
- $x_1 = -\eta_1 \cdot (z_1 - \text{sign}(z_1) \cdot \lambda_1) = -0.0091 \cdot (-490 - (-1) \cdot 0.1) = -0.0091 \cdot (-490 + 0.1) \approx -0.0091 \cdot (-489.9) \approx 4.46$
- New function value: $f(x_1) = 4.46^2 + 10 = 19.89 + 10 = 29.89$

### Iteration 2 (t = 2)
- Current position: $x_1 = 4.46$
- Gradient: $g_2 = f'(x_1) = 2 \times 4.46 = 8.92$
- Update sum of squared gradients: $\sigma_2 = \sigma_1 + g_2^2 = 100 + 8.92^2 = 100 + 79.57 = 179.57$
- Compute learning rate: $\eta_2 = \frac{\alpha}{\beta + \sqrt{\sigma_2}} = \frac{0.1}{1.0 + \sqrt{179.57}} = \frac{0.1}{1.0 + 13.4} \approx 0.0069$
- Update z value: $z_2 = z_1 + g_2 - \sigma_2 \cdot x_1 = -490 + 8.92 - 179.57 \times 4.46 = -490 + 8.92 - 800.88 = -1281.96$
- Since $|z_2| = 1281.96 > \lambda_1 = 0.1$, we update $x_2$:
- $x_2 = -\eta_2 \cdot (z_2 - \text{sign}(z_2) \cdot \lambda_1) = -0.0069 \cdot (-1281.96 - (-1) \cdot 0.1) \approx -0.0069 \cdot (-1281.86) \approx 8.84$
- New function value: $f(x_2) = 8.84^2 + 10 = 78.15 + 10 = 88.15$

### Iteration 3 (t = 3)
- Current position: $x_2 = 8.84$
- Gradient: $g_3 = f'(x_2) = 2 \times 8.84 = 17.68$
- Update sum of squared gradients: $\sigma_3 = \sigma_2 + g_3^2 = 179.57 + 17.68^2 = 179.57 + 312.58 = 492.15$
- Compute learning rate: $\eta_3 = \frac{\alpha}{\beta + \sqrt{\sigma_3}} = \frac{0.1}{1.0 + \sqrt{492.15}} = \frac{0.1}{1.0 + 22.18} \approx 0.0043$
- Update z value: $z_3 = z_2 + g_3 - \sigma_3 \cdot x_2 = -1281.96 + 17.68 - 492.15 \times 8.84 = -1281.96 + 17.68 - 4350.61 = -5614.89$
- Since $|z_3| = 5614.89 > \lambda_1 = 0.1$, we update $x_3$:
- $x_3 = -\eta_3 \cdot (z_3 - \text{sign}(z_3) \cdot \lambda_1) = -0.0043 \cdot (-5614.89 - (-1) \cdot 0.1) \approx -0.0043 \cdot (-5614.79) \approx 24.14$
- New function value: $f(x_3) = 24.14^2 + 10 = 582.74 + 10 = 592.74$

### Observations About FTRL Behavior

In this example, we can observe several interesting behaviors of FTRL:

1. **Divergence**: Unlike SGD, Adam, or Adamax, FTRL actually diverges in this example. This is because FTRL is primarily designed for problems with sparse features and L1 regularization, not for simple convex optimization problems like this one.

2. **Adaptive Learning Rate**: The learning rate decreases as more gradients are accumulated, but this doesn't help convergence in this case.

3. **The Role of L1 Regularization**: The L1 regularization term ($\lambda_1$) is meant to encourage sparsity, but in a one-dimensional problem like this, it doesn't have the same effect as in high-dimensional problems.

4. **The z Value**: The z value accumulates information about past gradients and helps determine the sign of the parameter update. In this case, it grows increasingly negative, leading to positive parameter updates.

### Modified Example with Better Parameters

Let's adjust our parameters to make FTRL work better for this simple function:

- Starting point: $x_0 = 5$
- Learning rate: $\alpha = 0.01$ (reduced)
- Beta: $\beta = 10.0$ (increased)
- L1 regularization: $\lambda_1 = 0.0$ (removed)
- L2 regularization: $\lambda_2 = 0.1$ (added)

With these parameters, FTRL would behave more like a standard adaptive gradient method and would converge to the minimum. The key changes are:
1. Reducing the learning rate
2. Removing L1 regularization (which is not needed for this problem)
3. Adding L2 regularization for stability
4. Increasing beta to slow down the initial steps

## Practical Considerations for FTRL

When using FTRL in practice, consider the following:

1. **Sparse Features**: FTRL works best with sparse feature spaces where feature selection is important.

2. **Hyperparameter Tuning**: The performance of FTRL is sensitive to hyperparameters, particularly the learning rate, beta, and regularization strengths.

3. **Online Learning**: FTRL is designed for online learning scenarios where data arrives sequentially.

4. **Memory Efficiency**: One of the main advantages of FTRL is its ability to produce sparse models, which can be memory-efficient.

5. **Convergence**: For simple convex optimization problems, other optimizers like Adam or SGD might converge faster and more reliably.

FTRL is a powerful optimizer for specific use cases, particularly in online advertising and recommendation systems where feature sparsity is important. However, it may not be the best choice for all types of optimization problems, especially simple convex problems like our example function.
