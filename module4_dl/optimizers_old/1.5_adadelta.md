# Adadelta Optimizer

Adadelta is an extension of Adagrad that seeks to address its diminishing learning rates problem. It was proposed by Matthew D. Zeiler in 2012 in his paper "ADADELTA: An Adaptive Learning Rate Method".

## Key Idea

The key innovation in Adadelta is replacing the accumulation of all past squared gradients with an exponentially decaying average of past squared gradients. Additionally, Adadelta eliminates the need for an initial learning rate parameter.

## Mathematical Formulation

The standard SGD update rule is:

$$\theta_{t+1} = \theta_t - \eta \cdot g_t$$

Where:
- $\theta_t$ represents the parameters at time step $t$
- $\eta$ is the learning rate
- $g_t$ is the gradient of the loss function with respect to the parameters at time step $t$

Adadelta modifies this update rule in two main ways:

### 1. Accumulating Squared Gradients with Decay

Instead of accumulating all past squared gradients like Adagrad, Adadelta uses an exponentially decaying average:

$$E[g^2]_t = \rho \cdot E[g^2]_{t-1} + (1 - \rho) \cdot g_t^2$$

Where:
- $E[g^2]_t$ is the expected value of the squared gradients up to time step $t$
- $\rho$ is the decay constant (typically set to around 0.95)

### 2. Accumulating Squared Parameter Updates

Adadelta also keeps track of the squared parameter updates:

$$E[\Delta \theta^2]_t = \rho \cdot E[\Delta \theta^2]_{t-1} + (1 - \rho) \cdot \Delta \theta_t^2$$

### 3. Parameter Update Rule

The parameter update is then computed as:

$$\Delta \theta_t = -\frac{\sqrt{E[\Delta \theta^2]_{t-1} + \epsilon}}{\sqrt{E[g^2]_t + \epsilon}} \cdot g_t$$
$$\theta_{t+1} = \theta_t + \Delta \theta_t$$

Where:
- $\epsilon$ is a small constant for numerical stability

## How Adadelta Improves on Adagrad and RMSProp

1. **No Learning Rate Parameter**: Unlike Adagrad and RMSProp, Adadelta doesn't require setting an initial learning rate.

2. **Prevents Diminishing Learning Rates**: By using an exponentially decaying average of squared gradients (like RMSProp), it prevents the aggressive learning rate decay seen in Adagrad.

3. **Units Correction**: By tracking the squared parameter updates, Adadelta ensures that the units of the parameter updates are consistent with the units of the parameters themselves.

## Advantages

1. **No Initial Learning Rate**: Eliminates the need to set an initial learning rate, reducing the number of hyperparameters to tune.

2. **Robust to Noisy Gradients**: The exponentially decaying average helps smooth out noise in the gradients.

3. **Adaptive Step Sizes**: Automatically adapts the step size for each parameter based on historical information.

4. **Theoretically Sound**: The units correction ensures that the parameter updates have the same units as the parameters.

## Disadvantages

1. **Computational Overhead**: Requires maintaining two moving averages instead of one.

2. **May Converge Slowly**: In some cases, Adadelta can converge more slowly than other adaptive methods like Adam.

3. **Additional Hyperparameter**: Introduces the decay rate $\rho$ as a hyperparameter to tune.

## When to Use Adadelta

Adadelta is particularly well-suited for:

- Situations where you want to avoid setting an initial learning rate
- Problems where Adagrad's aggressive learning rate decay is problematic
- Cases where you want theoretically sound parameter updates with proper units

## Comparison with Other Optimizers

- **vs. Adagrad**: Adadelta addresses Adagrad's diminishing learning rates problem by using an exponentially decaying average of past squared gradients.
  
- **vs. RMSProp**: Adadelta is similar to RMSProp in using an exponentially decaying average, but it also tracks squared parameter updates and eliminates the need for a learning rate.
  
- **vs. Adam**: Both are adaptive methods, but Adam incorporates momentum and bias correction, while Adadelta focuses on proper units and eliminating the learning rate parameter.

## Simple Example

Consider a parameter update with Adadelta:

1. Initialize $E[g^2]_0 = 0$ and $E[\Delta \theta^2]_0 = 0$
2. At time step $t$, compute gradient $g_t$
3. Update the running average of squared gradients: $E[g^2]_t = 0.95 \cdot E[g^2]_{t-1} + 0.05 \cdot g_t^2$
4. Compute the parameter update: $\Delta \theta_t = -\frac{\sqrt{E[\Delta \theta^2]_{t-1} + \epsilon}}{\sqrt{E[g^2]_t + \epsilon}} \cdot g_t$
5. Update the running average of squared parameter updates: $E[\Delta \theta^2]_t = 0.95 \cdot E[\Delta \theta^2]_{t-1} + 0.05 \cdot \Delta \theta_t^2$
6. Apply the update: $\theta_{t+1} = \theta_t + \Delta \theta_t$

This process continues iteratively, with the algorithm automatically adjusting the effective learning rate based on both gradient and update history.

## Numerical Example: Minimizing a Simple Function

Let's walk through a concrete example of using Adadelta to minimize the simple function:

$$f(x) = x^2 + 10$$

This is a simple parabola with a global minimum at $x = 0$, where $f(0) = 10$.

The derivative of this function is:

$$f'(x) = 2x$$

Now let's apply Adadelta to find the minimum, focusing on how the accumulation of squared parameter updates works:

### Initial Setup
- Starting point: $x_0 = 5$
- Decay rate: $\rho = 0.95$
- Small constant: $\epsilon = 1e-6$
- Accumulated squared gradient: $E[g^2]_0 = 0$
- Accumulated squared parameter updates: $E[\Delta x^2]_0 = 0$

### Iteration 1
- Current position: $x_0 = 5$
- Gradient: $g_1 = f'(x_0) = 2 \times 5 = 10$
- Accumulated squared gradient: $E[g^2]_1 = 0.95 \times 0 + 0.05 \times 10^2 = 0 + 5 = 5$
- Parameter update: $\Delta x_1 = -\frac{\sqrt{E[\Delta x^2]_0 + \epsilon}}{\sqrt{E[g^2]_1 + \epsilon}} \times g_1 = -\frac{\sqrt{0 + 1e-6}}{\sqrt{5 + 1e-6}} \times 10 \approx -\frac{0.001}{2.236} \times 10 \approx -0.0045$
- New position: $x_1 = x_0 + \Delta x_1 = 5 + (-0.0045) = 4.9955$
- Accumulated squared parameter updates: $E[\Delta x^2]_1 = 0.95 \times 0 + 0.05 \times (-0.0045)^2 = 0 + 0.00000101 \approx 0.000001$

### Iteration 2
- Current position: $x_1 = 4.9955$
- Gradient: $g_2 = f'(x_1) = 2 \times 4.9955 = 9.991$
- Accumulated squared gradient: $E[g^2]_2 = 0.95 \times 5 + 0.05 \times 9.991^2 = 4.75 + 4.99 = 9.74$
- Parameter update: $\Delta x_2 = -\frac{\sqrt{E[\Delta x^2]_1 + \epsilon}}{\sqrt{E[g^2]_2 + \epsilon}} \times g_2 = -\frac{\sqrt{0.000001 + 1e-6}}{\sqrt{9.74 + 1e-6}} \times 9.991 \approx -\frac{0.0014}{3.12} \times 9.991 \approx -0.0045$
- New position: $x_2 = x_1 + \Delta x_2 = 4.9955 + (-0.0045) = 4.991$
- Accumulated squared parameter updates: $E[\Delta x^2]_2 = 0.95 \times 0.000001 + 0.05 \times (-0.0045)^2 = 0.00000095 + 0.00000101 \approx 0.000002$

### Iteration 3
- Current position: $x_2 = 4.991$
- Gradient: $g_3 = f'(x_2) = 2 \times 4.991 = 9.982$
- Accumulated squared gradient: $E[g^2]_3 = 0.95 \times 9.74 + 0.05 \times 9.982^2 = 9.253 + 4.98 = 14.233$
- Parameter update: $\Delta x_3 = -\frac{\sqrt{E[\Delta x^2]_2 + \epsilon}}{\sqrt{E[g^2]_3 + \epsilon}} \times g_3 = -\frac{\sqrt{0.000002 + 1e-6}}{\sqrt{14.233 + 1e-6}} \times 9.982 \approx -\frac{0.0017}{3.77} \times 9.982 \approx -0.0045$
- New position: $x_3 = x_2 + \Delta x_3 = 4.991 + (-0.0045) = 4.9865$
- Accumulated squared parameter updates: $E[\Delta x^2]_3 = 0.95 \times 0.000002 + 0.05 \times (-0.0045)^2 = 0.0000019 + 0.00000101 \approx 0.000003$

### Key Observations About Squared Parameter Updates

Notice several important aspects of how the accumulated squared parameter updates work:

1. **Bootstrapping Problem**: In the first iteration, the accumulated squared parameter updates are zero, which would make the update zero. The $\epsilon$ term helps avoid this by providing a small initial value.

2. **Adaptive Step Size**: As the accumulated squared parameter updates grow, the step size adapts. This is particularly important when the gradients have different magnitudes across different dimensions.

3. **Unit Correction**: The numerator $\sqrt{E[\Delta x^2]_{t-1} + \epsilon}$ has the same units as the parameter $x$, while the denominator $\sqrt{E[g^2]_t + \epsilon}$ has the same units as the gradient. This ensures that the parameter update $\Delta x_t$ has the same units as $x$.

4. **No Learning Rate**: Notice that we didn't need to specify a learning rate. The ratio of the accumulated squared parameter updates to the accumulated squared gradients automatically determines an appropriate step size.

5. **Slow Initial Progress**: In the early iterations, the updates are small because the accumulated squared parameter updates are small. As training progresses, the updates can become larger as this accumulator grows.
