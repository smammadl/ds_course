# RMSProp Optimizer

RMSProp (Root Mean Square Propagation) is an optimization algorithm designed to address the diminishing learning rates problem of Adagrad. It was proposed by Geoffrey Hinton in his Neural Networks for Machine Learning course.

## Mathematical Formulation

The standard SGD update rule is:

$$\theta_{t+1} = \theta_t - \eta \cdot g_t$$

Where:
- $\theta_t$ represents the parameters at time step $t$
- $\eta$ is the learning rate
- $g_t$ is the gradient of the loss function with respect to the parameters at time step $t$

RMSProp modifies this update rule by using a moving average of squared gradients:

$$E[g^2]_t = \rho \cdot E[g^2]_{t-1} + (1 - \rho) \cdot g_t^2$$
$$\theta_{t+1} = \theta_t - \frac{\eta}{\sqrt{E[g^2]_t + \epsilon}} \cdot g_t$$

Where:
- $E[g^2]_t$ is the exponentially decaying average of squared gradients up to time step $t$
- $\rho$ is the decay rate (typically set to 0.9)
- $\epsilon$ is a small constant (typically on the order of 1e-8) added to avoid division by zero

## How RMSProp Improves on Adagrad

While Adagrad accumulates all past squared gradients, RMSProp uses an exponentially decaying average. This means:

1. Recent gradients have more influence than older gradients
2. The denominator doesn't grow indefinitely, preventing the learning rate from becoming infinitesimally small
3. The algorithm can continue learning even after many updates

## Advantages

1. **Adaptive Learning Rates**: Automatically adapts the learning rate for each parameter.

2. **Prevents Diminishing Learning Rates**: By using a moving average, it prevents the aggressive learning rate decay seen in Adagrad.

3. **Handles Non-Stationary Objectives**: Works well for non-convex optimization problems like neural networks.

4. **Effective for RNNs**: Particularly effective for Recurrent Neural Networks.

## Disadvantages

1. **Sensitive to Initial Learning Rate**: Still requires careful tuning of the initial learning rate.

2. **Additional Hyperparameter**: Introduces the decay rate $\rho$ as an additional hyperparameter to tune.

3. **No Momentum by Default**: The basic version doesn't include momentum, though it can be added.

## When to Use RMSProp

RMSProp is particularly well-suited for:

- Training deep neural networks
- Recurrent Neural Networks (RNNs)
- Non-convex optimization problems
- Situations where Adagrad's aggressive learning rate decay is problematic

RMSProp is often considered a good default choice for many deep learning applications, though Adam (which combines ideas from RMSProp and momentum) is now more commonly used.

## Understanding the Moving Average with a Numerical Example

Let's break down how the moving average of squared gradients works in RMSProp with a simple numerical example. Assume we're tracking the gradient for a single parameter with $\rho = 0.9$ (the typical value).

### Initial State
- Let's say $E[g^2]_0 = 0$ (we initialize the moving average to 0)

### Iteration 1
- We observe a gradient $g_1 = 10$
- $E[g^2]_1 = 0.9 \times 0 + 0.1 \times 10^2 = 0 + 10 = 10$

### Iteration 2
- We observe a gradient $g_2 = 5$
- $E[g^2]_2 = 0.9 \times 10 + 0.1 \times 5^2 = 9 + 2.5 = 11.5$

### Iteration 3
- We observe a gradient $g_3 = 8$
- $E[g^2]_3 = 0.9 \times 11.5 + 0.1 \times 8^2 = 10.35 + 6.4 = 16.75$

### Iteration 4
- We observe a gradient $g_4 = 2$
- $E[g^2]_4 = 0.9 \times 16.75 + 0.1 \times 2^2 = 15.075 + 0.4 = 15.475$

### Iteration 5
- We observe a gradient $g_5 = 1$
- $E[g^2]_5 = 0.9 \times 15.475 + 0.1 \times 1^2 = 13.9275 + 0.1 = 14.0275$

Notice how:
1. The moving average incorporates new gradient information (the squared gradient at each step)
2. But it gives more weight to recent history (90% of the previous average)
3. When gradients become smaller (iterations 4 and 5), the moving average gradually decreases
4. Unlike Adagrad, which would keep accumulating all squared gradients, RMSProp's moving average can both increase and decrease

This adaptive behavior allows RMSProp to maintain appropriate learning rates throughout training, even after many iterations.
