# Adagrad Optimizer

Adagrad (Adaptive Gradient Algorithm) is an optimization algorithm that adapts the learning rate to the parameters, performing smaller updates for parameters associated with frequently occurring features, and larger updates for parameters associated with infrequent features.

## Mathematical Formulation

The standard SGD update rule is:

$$\theta_{t+1} = \theta_t - \eta \cdot g_t$$

Where:
- $\theta_t$ represents the parameters at time step $t$
- $\eta$ is the learning rate
- $g_t$ is the gradient of the loss function with respect to the parameters at time step $t$

Adagrad modifies this update rule by introducing a parameter-specific learning rate:

$$\theta_{t+1,i} = \theta_{t,i} - \frac{\eta}{\sqrt{G_{t,ii} + \epsilon}} \cdot g_{t,i}$$

Where:
- $\theta_{t,i}$ is the $i$-th parameter at time step $t$
- $g_{t,i}$ is the gradient of the loss function with respect to $\theta_{t,i}$ at time step $t$
- $G_t$ is a diagonal matrix where each diagonal element $G_{t,ii}$ is the sum of the squares of the gradients with respect to $\theta_i$ up to time step $t$:

$$G_{t,ii} = \sum_{\tau=1}^{t} g_{\tau,i}^2$$

- $\epsilon$ is a small constant (typically on the order of 1e-8) added to avoid division by zero

## Advantages

1. **Adaptive Learning Rates**: Automatically adapts the learning rate for each parameter based on historical gradient information.

2. **Effective for Sparse Data**: Performs well on sparse data as it gives larger updates for infrequent parameters.

3. **No Manual Tuning of Learning Rate**: Reduces the need for manual tuning of the learning rate.

4. **Theoretical Guarantees**: Has strong theoretical guarantees for convex optimization problems.

## Disadvantages

1. **Accumulation of Squared Gradients**: The accumulated sum of squared gradients in the denominator continues to grow during training, causing the learning rate to shrink and eventually become infinitesimally small.

2. **Aggressive Learning Rate Decay**: The aggressive, monotonic decrease in learning rate can cause the algorithm to converge to suboptimal solutions for non-convex problems like neural networks.

3. **Uniform Scaling**: Treats all dimensions uniformly, which may not be optimal for all problems.

## When to Use Adagrad

Adagrad is particularly well-suited for:

- Problems with sparse features (e.g., natural language processing or computational advertising)
- Convex optimization problems
- Situations where you want to adapt the learning rate automatically for each parameter

However, for training deep neural networks, more advanced optimizers like Adam or RMSProp are often preferred due to Adagrad's aggressive learning rate decay.

## Simple Example: Minimizing a Scalar Function with Adagrad

Let's walk through a concrete example of using Adagrad to minimize the same simple scalar function we used for SGD:

$$f(x) = x^2 + 10$$

This is a simple parabola with a global minimum at $x = 0$, where $f(0) = 10$.

The derivative of this function is:

$$f'(x) = 2x$$

Now let's apply Adagrad to find the minimum:

### Initial Setup
- Starting point: $x_0 = 5$
- Base learning rate: $\eta = 0.1$
- Small constant: $\epsilon = 1e-8$
- Accumulated squared gradient: $G_0 = 0$
- We'll run for 5 iterations

### Iteration 1
- Current position: $x_0 = 5$
- Gradient: $g_1 = f'(x_0) = 2 \times 5 = 10$
- Accumulated squared gradient: $G_1 = G_0 + g_1^2 = 0 + 10^2 = 100$
- Adaptive learning rate: $\frac{\eta}{\sqrt{G_1 + \epsilon}} = \frac{0.1}{\sqrt{100 + 1e-8}} \approx \frac{0.1}{10} = 0.01$
- Update: $x_1 = x_0 - \frac{\eta}{\sqrt{G_1 + \epsilon}} \times g_1 = 5 - 0.01 \times 10 = 5 - 0.1 = 4.9$
- New function value: $f(x_1) = 4.9^2 + 10 = 24.01 + 10 = 34.01$

### Iteration 2
- Current position: $x_1 = 4.9$
- Gradient: $g_2 = f'(x_1) = 2 \times 4.9 = 9.8$
- Accumulated squared gradient: $G_2 = G_1 + g_2^2 = 100 + 9.8^2 = 100 + 96.04 = 196.04$
- Adaptive learning rate: $\frac{\eta}{\sqrt{G_2 + \epsilon}} = \frac{0.1}{\sqrt{196.04 + 1e-8}} \approx \frac{0.1}{14} \approx 0.0071$
- Update: $x_2 = x_1 - \frac{\eta}{\sqrt{G_2 + \epsilon}} \times g_2 = 4.9 - 0.0071 \times 9.8 = 4.9 - 0.07 = 4.83$
- New function value: $f(x_2) = 4.83^2 + 10 = 23.33 + 10 = 33.33$

### Iteration 3
- Current position: $x_2 = 4.83$
- Gradient: $g_3 = f'(x_2) = 2 \times 4.83 = 9.66$
- Accumulated squared gradient: $G_3 = G_2 + g_3^2 = 196.04 + 9.66^2 = 196.04 + 93.32 = 289.36$
- Adaptive learning rate: $\frac{\eta}{\sqrt{G_3 + \epsilon}} = \frac{0.1}{\sqrt{289.36 + 1e-8}} \approx \frac{0.1}{17} \approx 0.0059$
- Update: $x_3 = x_2 - \frac{\eta}{\sqrt{G_3 + \epsilon}} \times g_3 = 4.83 - 0.0059 \times 9.66 = 4.83 - 0.057 = 4.773$
- New function value: $f(x_3) = 4.773^2 + 10 = 22.78 + 10 = 32.78$

### Iteration 4
- Current position: $x_3 = 4.773$
- Gradient: $g_4 = f'(x_3) = 2 \times 4.773 = 9.546$
- Accumulated squared gradient: $G_4 = G_3 + g_4^2 = 289.36 + 9.546^2 = 289.36 + 91.13 = 380.49$
- Adaptive learning rate: $\frac{\eta}{\sqrt{G_4 + \epsilon}} = \frac{0.1}{\sqrt{380.49 + 1e-8}} \approx \frac{0.1}{19.5} \approx 0.0051$
- Update: $x_4 = x_3 - \frac{\eta}{\sqrt{G_4 + \epsilon}} \times g_4 = 4.773 - 0.0051 \times 9.546 = 4.773 - 0.049 = 4.724$
- New function value: $f(x_4) = 4.724^2 + 10 = 22.32 + 10 = 32.32$

### Iteration 5
- Current position: $x_4 = 4.724$
- Gradient: $g_5 = f'(x_4) = 2 \times 4.724 = 9.448$
- Accumulated squared gradient: $G_5 = G_4 + g_5^2 = 380.49 + 9.448^2 = 380.49 + 89.26 = 469.75$
- Adaptive learning rate: $\frac{\eta}{\sqrt{G_5 + \epsilon}} = \frac{0.1}{\sqrt{469.75 + 1e-8}} \approx \frac{0.1}{21.7} \approx 0.0046$
- Update: $x_5 = x_4 - \frac{\eta}{\sqrt{G_5 + \epsilon}} \times g_5 = 4.724 - 0.0046 \times 9.448 = 4.724 - 0.043 = 4.681$
- New function value: $f(x_5) = 4.681^2 + 10 = 21.91 + 10 = 31.91$

### Observations

Notice how:
1. The learning rate decreases with each iteration as the accumulated squared gradient increases
2. The progress toward the minimum is slower than with standard SGD (compare with the SGD example)
3. The step sizes become progressively smaller, which can be beneficial for fine-tuning but can also lead to very slow convergence
