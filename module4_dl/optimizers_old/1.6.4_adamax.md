# Adamax Optimizer

Adamax is a variant of the Adam optimizer introduced in the same paper by Kingma and Ba in 2014 ("Adam: A Method for Stochastic Optimization"). It is a modification of Adam that uses the infinity norm instead of the L2 norm for the second moment estimate.

## Key Idea

The key insight behind Adamax is to replace the second moment estimate (which uses the L2 norm in Adam) with an exponentially weighted infinity norm. This makes the algorithm more robust to gradient updates with large magnitudes and potentially more stable in some scenarios.

## Mathematical Formulation

The standard Adam update rule uses the L2 norm for the second moment estimate. Adamax modifies this by using the $L_{\infty}$ norm instead.

### 1. Compute Momentum (First Moment)

$$m_t = \beta_1 \cdot m_{t-1} + (1 - \beta_1) \cdot g_t$$

Where:
- $m_t$ is the estimate of the first moment (mean) of the gradients
- $\beta_1$ is the exponential decay rate for the first moment estimates (typically 0.9)
- $g_t$ is the gradient at time step $t$

### 2. Compute Infinity Norm Update (Instead of RMSProp)

$$u_t = \max(\beta_2 \cdot u_{t-1}, |g_t|)$$

Where:
- $u_t$ is the exponentially weighted infinity norm of the gradients
- $\beta_2$ is the exponential decay rate for the infinity norm (typically 0.999)
- $|g_t|$ is the absolute value of the gradient

This is the key difference from Adam, which uses:
$$v_t = \beta_2 \cdot v_{t-1} + (1 - \beta_2) \cdot g_t^2$$

### 3. Bias Correction (First Moment Only)

$$\hat{m}_t = \frac{m_t}{1 - \beta_1^t}$$

Note that unlike Adam, Adamax doesn't require bias correction for the infinity norm estimate $u_t$.

### 4. Parameter Update

$$\theta_{t+1} = \theta_t - \frac{\eta}{\hat{u}_t} \cdot \hat{m}_t = \theta_t - \frac{\eta}{u_t} \cdot \hat{m}_t$$

Where:
- $\theta_t$ represents the parameters at time step $t$
- $\eta$ is the learning rate
- $\hat{m}_t$ is the bias-corrected first moment estimate
- $u_t$ is the infinity norm estimate

## The Infinity Norm Advantage

The use of the infinity norm in Adamax provides several potential advantages:

1. **Robustness to Outliers**: The infinity norm is less sensitive to extreme gradient values than the L2 norm used in Adam.

2. **Simpler Update Rule**: Adamax doesn't require the square root operation that Adam does, which can be computationally beneficial.

3. **Theoretical Stability**: In some cases, the infinity norm can provide more stable updates, especially when gradients have high variance.

## Advantages of Adamax

1. **Stability**: Adamax can be more stable than Adam in the presence of large gradients or noisy gradients.

2. **Simplicity**: The update rule is simpler than Adam's as it doesn't require the square root operation.

3. **Robustness**: The infinity norm makes Adamax more robust to gradient outliers.

4. **Consistent Performance**: Adamax often performs more consistently across different problems and hyperparameter settings.

## When to Use Adamax

Adamax is particularly well-suited for:

- Problems with sparse gradients
- When Adam exhibits instability
- When you observe large gradient values or gradient explosions
- When you want a potentially more robust alternative to Adam with minimal parameter tuning

## Numerical Example: Minimizing a Simple Function with Adamax

Let's walk through a concrete example of using Adamax to minimize the simple function:

$$f(x) = x^2 + 10$$

This is a simple parabola with a global minimum at $x = 0$, where $f(0) = 10$.

The derivative of this function is:

$$f'(x) = 2x$$

Now let's apply Adamax to find the minimum:

### Initial Setup
- Starting point: $x_0 = 5$
- Learning rate: $\eta = 0.1$
- First moment decay rate: $\beta_1 = 0.9$
- Second moment decay rate: $\beta_2 = 0.999$
- First moment: $m_0 = 0$
- Infinity norm: $u_0 = 0$
- Time step: $t = 0$

### Iteration 1 (t = 1)
- Current position: $x_0 = 5$
- Gradient: $g_1 = f'(x_0) = 2 \times 5 = 10$
- Update first moment: $m_1 = \beta_1 \times m_0 + (1 - \beta_1) \times g_1 = 0.9 \times 0 + 0.1 \times 10 = 1$
- Update infinity norm: $u_1 = \max(\beta_2 \times u_0, |g_1|) = \max(0.999 \times 0, |10|) = \max(0, 10) = 10$
- Bias-corrected first moment: $\hat{m}_1 = \frac{m_1}{1 - \beta_1^1} = \frac{1}{1 - 0.9} = 10$
- Update: $x_1 = x_0 - \frac{\eta \times \hat{m}_1}{u_1} = 5 - \frac{0.1 \times 10}{10} = 5 - 0.1 = 4.9$
- New function value: $f(x_1) = 4.9^2 + 10 = 24.01 + 10 = 34.01$

### Iteration 2 (t = 2)
- Current position: $x_1 = 4.9$
- Gradient: $g_2 = f'(x_1) = 2 \times 4.9 = 9.8$
- Update first moment: $m_2 = \beta_1 \times m_1 + (1 - \beta_1) \times g_2 = 0.9 \times 1 + 0.1 \times 9.8 = 0.9 + 0.98 = 1.88$
- Update infinity norm: $u_2 = \max(\beta_2 \times u_1, |g_2|) = \max(0.999 \times 10, |9.8|) = \max(9.99, 9.8) = 9.99$
- Bias-corrected first moment: $\hat{m}_2 = \frac{m_2}{1 - \beta_1^2} = \frac{1.88}{1 - 0.81} = \frac{1.88}{0.19} = 9.89$
- Update: $x_2 = x_1 - \frac{\eta \times \hat{m}_2}{u_2} = 4.9 - \frac{0.1 \times 9.89}{9.99} = 4.9 - 0.099 = 4.801$
- New function value: $f(x_2) = 4.801^2 + 10 = 23.05 + 10 = 33.05$

### Iteration 3 (t = 3)
- Current position: $x_2 = 4.801$
- Gradient: $g_3 = f'(x_2) = 2 \times 4.801 = 9.602$
- Update first moment: $m_3 = \beta_1 \times m_2 + (1 - \beta_1) \times g_3 = 0.9 \times 1.88 + 0.1 \times 9.602 = 1.692 + 0.9602 = 2.6522$
- Update infinity norm: $u_3 = \max(\beta_2 \times u_2, |g_3|) = \max(0.999 \times 9.99, |9.602|) = \max(9.98, 9.602) = 9.98$
- Bias-corrected first moment: $\hat{m}_3 = \frac{m_3}{1 - \beta_1^3} = \frac{2.6522}{1 - 0.729} = \frac{2.6522}{0.271} = 9.79$
- Update: $x_3 = x_2 - \frac{\eta \times \hat{m}_3}{u_3} = 4.801 - \frac{0.1 \times 9.79}{9.98} = 4.801 - 0.098 = 4.703$
- New function value: $f(x_3) = 4.703^2 + 10 = 22.12 + 10 = 32.12$

### Key Observations About Adamax

Notice several important aspects of how Adamax works:

1. **Stable Updates**: Adamax provides stable updates due to the infinity norm, which is less affected by large gradient values.

2. **Consistent Step Sizes**: The step sizes in Adamax tend to be more consistent across iterations compared to Adam, especially when gradients vary significantly in magnitude.

3. **Slower Initial Progress**: Compared to SGD, Adamax (like Adam) makes slower initial progress but is more stable and less likely to overshoot.

4. **Infinity Norm Behavior**: The infinity norm $u_t$ tends to be dominated by the largest gradient magnitude seen so far, scaled by $\beta_2^k$ where $k$ is the number of steps since that large gradient was observed.

## Practical Considerations

When using Adamax in practice, consider the following:

1. **Learning Rate**: Adamax typically works well with a slightly higher learning rate than Adam (e.g., 0.002 instead of 0.001).

2. **Decay Rates**: The default values of $\beta_1 = 0.9$ and $\beta_2 = 0.999$ work well in most cases.

3. **Sparse Gradients**: Adamax can be particularly effective for problems with sparse gradients.

4. **Noisy Gradients**: If your problem has noisy gradients with occasional large values, Adamax might provide more stable updates than Adam.

Adamax is a robust alternative to Adam that can provide more stable optimization in certain scenarios, particularly when dealing with gradients that have high variance or occasional large values.
