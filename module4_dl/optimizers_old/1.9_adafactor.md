# Adafactor Optimizer

Adafactor is a memory-efficient adaptive optimization algorithm introduced by Shazeer and Stern in their 2018 paper "Adafactor: Adaptive Learning Rates with Sublinear Memory Cost". It was designed specifically to address the memory constraints when training very large neural networks, particularly Transformer models.

## Key Idea

The key insight behind Adafactor is to reduce the memory footprint of adaptive optimizers like Adam by:

1. Factorizing the second moment estimates (squared gradients) into row and column factors
2. Using a moving average of the squared gradients instead of storing the full history
3. Incorporating techniques to reduce the need for additional optimizer state

This makes Adafactor particularly valuable for training large language models and other parameter-heavy neural networks where memory is a constraint.

## Memory Efficiency

For a model with parameters in an $m \times n$ matrix, Adam requires storing two full $m \times n$ matrices for the first and second moments. In contrast, Adafactor only needs to store:

- Row sums: An $m$-dimensional vector
- Column sums: An $n$-dimensional vector
- Scalar statistics: A few scalar values

This reduces the memory requirement from $O(mn)$ to $O(m+n)$, which is a significant saving for large matrices.

## Mathematical Formulation

The Adafactor update rule involves several steps:

### 1. Compute Factorized Second Moment Estimates

For a parameter matrix $W \in \mathbb{R}^{m \times n}$ with gradient $G$:

$$r_t = \beta_2 \cdot r_{t-1} + (1 - \beta_2) \cdot \text{rowsum}(G_t^2)$$
$$c_t = \beta_2 \cdot c_{t-1} + (1 - \beta_2) \cdot \text{colsum}(G_t^2)$$

Where:
- $r_t \in \mathbb{R}^m$ is the vector of row sums
- $c_t \in \mathbb{R}^n$ is the vector of column sums
- $\beta_2$ is the decay rate (typically 0.999)
- $\text{rowsum}$ and $\text{colsum}$ compute the sum of squares along rows and columns

### 2. Approximate the Second Moment Matrix

$$V_t(i,j) \approx \frac{r_t(i) \cdot c_t(j)}{\sqrt{\sum_i r_t(i) \cdot \sum_j c_t(j)}}$$

This approximation assumes that the second moment matrix can be factorized, which works well in practice.

### 3. Compute the Update

$$W_{t+1} = W_t - \alpha_t \cdot G_t / \sqrt{V_t + \epsilon}$$

Where:
- $\alpha_t$ is the learning rate at time $t$
- $\epsilon$ is a small constant for numerical stability

### 4. Learning Rate Scheduling

Adafactor often uses a learning rate schedule based on the step count:

$$\alpha_t = \min(\alpha_{\text{base}}, \alpha_{\text{base}} \cdot \text{decay\_rate}^{-0.5+0.5 \cdot \text{warmup\_steps} / \max(t, \text{warmup\_steps})})$$

This schedule provides a warmup period followed by decay.

## Advantages of Adafactor

1. **Memory Efficiency**: Requires significantly less memory than Adam, making it suitable for very large models.

2. **Adaptive Learning Rates**: Like Adam, it adapts learning rates based on historical gradient information.

3. **Built-in Learning Rate Scheduling**: Includes relative step size adaptation, reducing the need for manual learning rate scheduling.

4. **Stability**: Often provides stable training for Transformer models and other deep networks.

5. **Practical for Large Models**: Has been successfully used to train some of the largest language models.

## When to Use Adafactor

Adafactor is particularly well-suited for:

- Training very large models where memory is a constraint
- Transformer-based architectures (e.g., BERT, T5, GPT)
- When you need adaptive learning rates but can't afford Adam's memory footprint
- Fine-tuning large pre-trained models on limited hardware

## Numerical Example: Minimizing a Simple Function with Adafactor

Let's walk through a concrete example of using Adafactor to minimize the simple function:

$$f(x) = x^2 + 10$$

This is a simple parabola with a global minimum at $x = 0$, where $f(0) = 10$.

The derivative of this function is:

$$f'(x) = 2x$$

For this one-dimensional problem, the factorization aspect of Adafactor doesn't come into play, but we can still demonstrate its adaptive learning rate behavior.

### Initial Setup
- Starting point: $x_0 = 5$
- Base learning rate: $\alpha_{\text{base}} = 0.1$
- Second moment decay rate: $\beta_2 = 0.999$
- Epsilon: $\epsilon = 1e-6$
- Initial second moment estimate: $v_0 = 0$
- Time step: $t = 0$
- Relative step size: True (using step size adaptation)

### Iteration 1 (t = 1)
- Current position: $x_0 = 5$
- Gradient: $g_1 = f'(x_0) = 2 \times 5 = 10$
- Update second moment estimate: $v_1 = \beta_2 \times v_0 + (1 - \beta_2) \times g_1^2 = 0.999 \times 0 + 0.001 \times 10^2 = 0.1$
- Compute relative learning rate: $\alpha_1 = \min(0.1, 0.1 / \sqrt{1}) = 0.1$
- Update: $x_1 = x_0 - \alpha_1 \times g_1 / \sqrt{v_1 + \epsilon} = 5 - 0.1 \times 10 / \sqrt{0.1 + 1e-6} \approx 5 - 0.1 \times 10 / 0.316 \approx 5 - 3.16 = 1.84$
- New function value: $f(x_1) = 1.84^2 + 10 = 3.39 + 10 = 13.39$

### Iteration 2 (t = 2)
- Current position: $x_1 = 1.84$
- Gradient: $g_2 = f'(x_1) = 2 \times 1.84 = 3.68$
- Update second moment estimate: $v_2 = \beta_2 \times v_1 + (1 - \beta_2) \times g_2^2 = 0.999 \times 0.1 + 0.001 \times 3.68^2 = 0.0999 + 0.0135 = 0.1134$
- Compute relative learning rate: $\alpha_2 = \min(0.1, 0.1 / \sqrt{2}) = 0.0707$
- Update: $x_2 = x_1 - \alpha_2 \times g_2 / \sqrt{v_2 + \epsilon} = 1.84 - 0.0707 \times 3.68 / \sqrt{0.1134 + 1e-6} \approx 1.84 - 0.0707 \times 3.68 / 0.337 \approx 1.84 - 0.77 = 1.07$
- New function value: $f(x_2) = 1.07^2 + 10 = 1.14 + 10 = 11.14$

### Iteration 3 (t = 3)
- Current position: $x_2 = 1.07$
- Gradient: $g_3 = f'(x_2) = 2 \times 1.07 = 2.14$
- Update second moment estimate: $v_3 = \beta_2 \times v_2 + (1 - \beta_2) \times g_3^2 = 0.999 \times 0.1134 + 0.001 \times 2.14^2 = 0.1133 + 0.0046 = 0.1179$
- Compute relative learning rate: $\alpha_3 = \min(0.1, 0.1 / \sqrt{3}) = 0.0577$
- Update: $x_3 = x_2 - \alpha_3 \times g_3 / \sqrt{v_3 + \epsilon} = 1.07 - 0.0577 \times 2.14 / \sqrt{0.1179 + 1e-6} \approx 1.07 - 0.0577 \times 2.14 / 0.343 \approx 1.07 - 0.36 = 0.71$
- New function value: $f(x_3) = 0.71^2 + 10 = 0.50 + 10 = 10.50$

### Key Observations About Adafactor

Notice several important aspects of how Adafactor works:

1. **Faster Initial Convergence**: In this simple example, Adafactor converges faster initially than both SGD and Adam due to its adaptive step size.

2. **Decreasing Step Sizes**: The relative step size decreases with the square root of the iteration count, which helps stabilize later training.

3. **Adaptive Learning Rate**: The effective learning rate is adjusted based on the second moment of gradients, similar to Adam but with less memory overhead.

4. **Simplicity in 1D**: For this one-dimensional problem, Adafactor's factorization doesn't provide memory benefits, but in high-dimensional problems, the savings would be substantial.

## Practical Considerations for Adafactor

When using Adafactor in practice, consider the following:

1. **Matrix Shapes**: Adafactor works best when parameters are organized in matrices where factorization makes sense.

2. **Learning Rate**: The default relative step size often works well, but you may need to adjust the base learning rate.

3. **Warmup**: For deep networks, a warmup period can be beneficial.

4. **Clipping**: Adafactor includes gradient clipping by default, which helps stability.

5. **Momentum**: Unlike Adam, the standard Adafactor doesn't use momentum for the first moment, though variants exist that incorporate it.

Adafactor represents an important advancement in optimization algorithms, particularly for large-scale deep learning where memory efficiency is crucial. Its ability to provide adaptive learning rates with significantly reduced memory requirements makes it an essential tool for training very large models on limited hardware.
